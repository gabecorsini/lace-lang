Language Reference

Lace is a statically-typed, effect-annotated language designed for agentic execution. Programs run on a lean interpreter or bytecode VM. The core design principle: make failure modes loud at compile time, not silent at runtime.

Option<T>Present or absent valueSome(x), None
Result<T, E>Success or failureOk(x), Err(e)
UnitNo meaningful value(implicit)
RecordsNamed field structsrecord Point { x: Int, y: Int }

There are no nulls in Lace. Absent values are always Option<T>.

  print("positive")

Functions

fn add(a: Int, b: Int) -> Int {
  a + b
}

# Closures
let double = fn(x: Int) { x * 2 }

# Higher-order
fn apply(f: fn(Int) -> Int, x: Int) -> Int {
  f(x)
}

fn declares a pure function — no I/O, no side effects. The type checker warns (W004) if a fn calls effectful operations. If you need I/O, use tool.

Tools & Effects

@retry(max: 3)
tool fetch_user(id: Int) -> Result<String, String> {
  let body = Http.get("https://api.example.com/users/" ++ id.to_string())?
  let data = Json.parse(body)?
  Json.get(data, "name") |> Option.unwrap_or("unknown") |> Ok
}

Decorators

DecoratorDescription
@retry(max: N)Retry the tool up to N times on Err. Logs each retry attempt.
@timeout(ms: N)Emit a warning if the tool takes longer than N milliseconds.

Automatic Tool Logging

Every tool call automatically emits JSON to stderr:

{"event":"tool_call","tool":"fetch_user","args":[42],"timestamp":1716000000000}
{"event":"tool_ok","tool":"fetch_user","duration_ms":120}
{"event":"tool_err","tool":"fetch_user","error":"connection refused","duration_ms":5}
{"event":"tool_retry","tool":"fetch_user","attempt":2}

Suppress with lace run --no-tool-log.

Error Handling

Errors are values. No exceptions.

# Return Ok or Err
fn divide(a: Int, b: Int) -> Result<Int, String> {
  if b == 0 {
    Err("division by zero")
  } else {
    Ok(a // b)
  }
}

# ? propagates Err up the call stack
fn compute(a: Int, b: Int) -> Result<Int, String> {
  let result = divide(a, b)?   # returns Err early if divide fails
  Ok(result * 2)
}

Modules

import utils         # imports utils.lace from same directory
import lib/helpers   # imports lib/helpers.lace

Module resolution uses the entry field in lace.toml as the root. All imports are relative to the project root.

Grammar (EBNF)

program     = { top_level } ;
top_level   = fn_decl | tool_decl | record_decl | import_stmt | const_decl ;

fn_decl     = { decorator } "fn" IDENT "(" params ")" "->" type block ;
tool_decl   = { decorator } "tool" IDENT "(" params ")" "->" type block ;
decorator   = "@" IDENT "(" decorator_args ")" ;

params      = [ param { "," param } ] ;
param       = IDENT ":" type ;

type        = "Int" | "Float" | "Bool" | "String" | "Unit"
            | "List" "<" type ">"
            | "Map" "<" type "," type ">"
            | "Option" "<" type ">"
            | "Result" "<" type "," type ">"
            | IDENT ;

block       = "{" { stmt } "}" ;
stmt        = let_stmt | assign_stmt | expr_stmt | return_stmt
            | if_stmt | while_stmt | for_stmt | match_stmt ;

let_stmt    = "let" [ "mut" ] IDENT [ ":" type ] "=" expr ";" ;
return_stmt = "return" [ expr ] ";" ;
expr_stmt   = expr ";" ;

expr        = binary | unary | call | index | field
            | closure | literal | IDENT | "(" expr ")" ;
binary      = expr op expr ;
op          = "+" | "-" | "*" | "/" | "//" | "%" | "++"
            | "==" | "!=" | "<" | "<=" | ">" | ">="
            | "&&" | "||" | "|>" ;
call        = expr "(" args ")" [ "?" ] ;

literal     = INT | FLOAT | STRING | "true" | "false"
            | "[" [ expr { "," expr } ] "]"
            | "{" [ STRING ":" expr { "," STRING ":" expr } ] "}" ;
