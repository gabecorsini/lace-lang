CLI Reference

The lace CLI is the primary interface for working with Lace programs — running, checking, formatting, building docs, and serving the LSP.

Quick Start

cargo build --workspace
./target/debug/lace new myproject
cd myproject
./target/debug/lace run src/main.lace

lace run

Run a .lace source file through the interpreter (or bytecode VM with --vm).

lace run <file> [--no-warn] [--no-tool-log] [--vm]
FlagDescription
--no-warnSuppress all warnings (W001–W004)
--no-tool-logSuppress automatic JSON tool call logs on stderr
--vmUse the bytecode VM instead of tree-walking interpreter
lace run src/main.lace
lace run src/main.lace --no-tool-log
lace run src/main.lace --vm

lace check

Type-check a file without running it. Prints all errors and warnings, exits non-zero on errors.

lace check <file> [--no-warn]
lace check src/main.lace

lace build

Type-check the whole project defined in lace.toml. Reports file count and item count.

lace build [--no-warn]
lace build
# checked 3 files, 12 items — no errors

lace fmt

Format a .lace file in place. Idempotent — formatted files are unchanged on re-run.

lace fmt <file>
lace fmt src/main.lace

lace doc

Generate HTML documentation from ## doc comments. Outputs to docs/.

lace doc [path]
lace doc examples/
lace doc src/main.lace

lace explain

Print a detailed explanation of an error or warning code.

lace explain <code>
lace explain E001
lace explain W004

lace lsp

Start the LSP server on stdio. Editors invoke this as a subprocess — you don't run it directly.

lace lsp

See the LSP & Editor Setup page for configuration instructions.

lace new

Scaffold a new Lace project with lace.toml and src/main.lace.

lace new <name>
lace new myproject
# Created myproject/lace.toml
# Created myproject/src/main.lace

lace compile

Compile a .lace file to a .lacec bytecode file in the same directory.

lace compile <file>
lace compile src/main.lace
# compiled: src/main.lacec

lace vm

Run a .lacec bytecode file directly. Also accepts .lace (compile + run in one step).

lace vm <file>
lace vm src/main.lacec
lace vm src/main.lace   # compiles on the fly

Error & Warning Codes

CodeNameDescription
E001UnknownIdentifierVariable or function not found in scope. Triggers "did you mean?" suggestions.
E002TypeMismatchExpression type doesn't match expected type.
E003ArityMismatchWrong number of arguments to a function or tool call.
E004NotCallableAttempted to call something that isn't a function or tool.
E005MissingFieldRecord field access on a field that doesn't exist.
W001UnusedVariableVariable declared but never read.
W002UnreachableCodeCode after an unconditional return.
W003reserved—
W004PureFnCallsEffectfulA fn calls a tool or effectful stdlib function. Consider declaring it as a tool.

lace.toml

Project manifest file. Place in the project root.

[project]
name = "myproject"
version = "0.1.0"
entry = "src/main.lace"
FieldDescription
nameProject name (used in doc output and bundle metadata)
versionSemver string
entryPath to the main entry file, relative to project root
